# PR 생성 스킬

## 목적
GitHub Pull Request를 자동으로 생성한다.

## 규칙

다음 규칙을 반드시 지켜 GitHub PR을 생성한다.

### PR 제목 규칙
- 형식: 🔀 :: (#이슈번호) - 제목
- 반드시 한글로 작성
- 간결하고 작업 핵심만 포함

### 본문 규칙
- 제공된 PR 템플릿을 그대로 사용
- 모든 항목은 한글로 작성
- "관련 이슈"에는 Closes #{이슈번호} 자동 삽입
- 작업내용은 bullet point로 구체적으로 작성
- 테스트 방법은 실제 실행 기준으로 작성
- PR 본문 파일은 반드시 UTF-8로 저장
- PowerShell에서 템플릿을 읽을 때는 `-Encoding utf8` 명시
- GitHub CLI 사용 시 본문은 `--body-file`로 전달

### 추가 규칙
- 영어 사용 금지 (코드 제외)
- 불필요한 문장 금지
- 리뷰어가 이해하기 쉽게 작성

## 동작

1. 현재 브랜치명에서 이슈 번호를 추출한다. (예: #123)
2. 커밋 메시지를 기반으로 작업 내용을 요약한다.
3. PR 제목을 생성한다.
4. PR 템플릿을 기반으로 본문을 생성한다.
5. GitHub PR을 생성한다.
